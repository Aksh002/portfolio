### Problem

Prompt-to-code generation becomes risky when the output is executable animation code. The challenge was to make the system useful, asynchronous, and safe enough for iterative rendering.

### Approach

I designed the platform as a modular render pipeline rather than a single prompt endpoint:

- exposed dedicated APIs for generate, render, status, video delivery, and regeneration flows
- used Redis and RQ to orchestrate async render jobs and persistent job-state tracking
- added AST validation and sandboxed execution with scene checks, unsafe import blocking, and Docker resource limits
- built an error-feedback retry loop plus a Monaco-powered interface for prompt edits and re-rendering

### Outcome

The result is a full-stack educational video generation product with safer execution, clearer debugging, and a significantly better iteration loop for generated Manim code.

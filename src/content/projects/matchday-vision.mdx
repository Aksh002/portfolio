### Problem

Payment products need secure authentication, clean service boundaries, and reliable transaction handling, especially when user and merchant experiences evolve in parallel.

### Approach

I built the system like a modular product suite with backend-first thinking:

- used Turborepo to model boundaries between payments, auth, and transaction services
- implemented multi-provider authentication with NextAuth.js
- designed asynchronous payment flows using Express.js webhooks and job queues
- Dockerized services and wired CI/CD-oriented deployment for scalability and minimal downtime

### Outcome

The result is a secure P2P and P2M payments platform that demonstrates monorepo architecture, full-stack coordination, and production-minded backend design.
